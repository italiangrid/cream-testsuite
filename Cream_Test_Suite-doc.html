TEST CASE: Enable - Disable Submission
Full Name:Cream Test Suite.Enable - Disable Submission
Documentation:This test case tests the proper functionality of the glite-ce-allowed-submission and glite-ce-enable/disable-submission commands, for admin and non-admin users.
The test executes the following steps:
1) Checks if initially the submission is allowed (it should)
2) Executes a simple job (it should succeed)
3) Disables submission (the current user executing the test case is considered to NOT be a CREAM admin, thus this step should fail)
4) Add the current user as a CREAM admin
5) Disable submission (this time it should succeed)
6) Check if submission is allowed (it shouldn't)
7) Execute a simple job submission (it should fail)
8) Enable submission
9) Execute a simple job (it should succeed)
10) Remove current user from CREAM admin
11) Disable submission (should fail, since the user isn't a CREAM admin anymore)
This test case internally uses root ssh access to the cream host, in order to add/remove a cream administrator.
Tags:en_dis_sub

Full Name:Cream Test Suite.Admin Manage Job
Documentation:This test case checks whether a user can manage other people's jobs when being or not being an admin. Test case algorithm:
1) Submit a sleep job and wait until it reaches the REALLY-RUNNING state
2) Create a new proxy with different credentials that the default (a second cert and key are required for this, pointed to by the relevant variables)
3) Try to cancel the job (should fail, since the user with the second proxy isn't an admin)
4) Make the user with the second proxy a cream admin
5) Try to cancel the job (should succeed, since the user with the second proxy is an admin now)
6) Remove the user with the second proxy from the admin list.
